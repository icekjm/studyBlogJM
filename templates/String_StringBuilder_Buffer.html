String, StringBuffer, StringBuilder 차이

  String 객체는 한번 생성되면 불변적인 특징 때문에 값을 업데이트하면, 매 연산 시마다 새로운 문자열을 가진 String 인스턴스가 생성되어 메모리공간을 차지하게 되지만, StringBuffer와 StringBuilder 는 가변성 가지기 때문에 .append() .delete() 등의 API를 이용하여 동일 객체내에서 문자열 크기를 변경하는 것이 가능하다.



  따라서 값이 변경될 때마다 새롭게 객체를 만드는 String 보다 훨씬 빠르기 때문에, 문자열의 추가, 수정, 삭제가 빈번하게 발생할 경우라면 String 클래스가 아닌 StringBuffer 또는 StringBuilder를 사용하는 것이 이상적이라 말할 수 있다.
  StringBuffer와 StringBuilder의 차이점은, 멀티 쓰레드(Thread)에서 안전(safe)하냐 아니냐 이 차이 뿐이다. 결론적으로, StringBuffer는 멀티스레드 환경에서 안전하고, StringBuilder는 멀티스레드 환경에서 불안정하다.
  그 이유는, 아래 그림처럼 하나의 StringBuilder객체에 3개의 스레드들이 접근하는 중이다. 이들을 사람으로 비유해서, 각 사람이 하나의 버튼을 한번씩 누른다고 가정하자. 그러면 총 3번의 버튼이 눌려야 하지만 실제로는 그렇지 않다. 왜냐하면 세 사람 전부 동시에 한번씩만 누를수 있기 때문이다.
  따라서, StringBuffer의 경우, 누군가 누르고 있으면 기다렸다가 앞의 사람이 다 누른뒤에 그 다음 사람이 버튼을 누르게 된다. 이거를 동기화라고 하며 이러한 동기화때문에 StringBuffer는 멀티스레드 환경에서 안전하다.
